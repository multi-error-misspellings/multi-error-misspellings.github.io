CLIQUE Project
  1. The idea of CLIQUE is to help users to search through NASA Earth Data , even when committing misspellings.

  2. It is sometimes the problem of a misspelling that makes a dataset unidentified during the search.

  3. Some users are nonnative English speakers. They might want to search for a dataset but they only remember some parts of the word.

  4. CLIQUE allow multiple errors in a word, upto 5 errors. For example, if searching for atmosphere, a user entered atmusfer.

Notes:
  • Phonetic error: misspelling error that results into a similar sounded word, like fonetik for phonetic. Usually this error category is highly committed by a large number of users, especially dyslexic users.
  • Misconceived error: misspelling error that might sound and look the same as the correct word caused due to lack of knowledge of correct word, like anonomy for anatomy. Usually this error category is caused by nonnative speakers of English.
  • Keyboard error: misspelling error that is caused by fast keyboard typing, like amification for amplification. Also, the fat-finger syndrome where the user hit the near-by key instead of correct key, like leatn for learn â€“ where t-letter is near r-letter using QWERTY keyboard.

Clique is tested using the benchmark dataset of Hunspell and Aspell spell checkers. The benchmark dataset is optimally-sized, yet it challenges any spell checker software: the dataset quality is high due to its comprehension and diversity in error operations and categories.

See: Leena Al-Hussaini, Experience: Insights into the Benchmarking Data of Hunspell and Aspell Spell Checkers , published in ACM Journal of Data and Information Quality, June 2017, Article No.: 13.
